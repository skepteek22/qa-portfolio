# [BUG-001] UI: Inconsistent design of main menu category icons

**Status:** 🔴 Open  
**Severity:** S3 (Major)  
**Priority:** P3 (Low)  
**Project:** DemoQA  

---

## 📝 Description
The icons within the main navigation category tiles on the home page do not follow a unified design pattern. Specifically, icons for **"Elements"** and **"Interactions"** appear visually larger or have different padding/scaling compared to **"Forms"** and **"Widgets"**.

## 💻 Environment
- **URL:** [https://demoqa.com/](https://demoqa.com/)
- **Browser:** Chrome (Latest version)
- **Device:** Desktop / Laptop
- **Resolution:** 1920x1080

## 🐾 Steps to Reproduce
1. Navigate to the `demoqa.com` home page.
2. Scroll down to the category tiles section (middle of the page).
3. Visually compare the blue circular icons for each category tile.

## ❌ Actual Result
Icons have inconsistent dimensions and scaling. For example, the **"Interactions"** icon is visually larger than the **"Forms"** icon, leading to an unpolished and "messy" UI.

## ✅ Expected Result
All main menu icons should have identical dimensions, consistent scaling, and uniform padding within their circular containers according to the UI style guide.
